## ACL (Access Control List) 정리 - 공식 패턴

---

## 1. 인터페이스 적용 기본 패턴

```bash
interface [인터페이스]
 ip access-group [ACL번호] [in | out]

# 예시
interface s1/0
 ip access-group 10 out

# 의미
S1/0 인터페이스에서 나가는(out) 트래픽을 ACL 10으로 필터링
```

## 2. Standard ACL (1~99)
```
# 기본 구조
access-list [번호] permit|deny [출발지] [와일드카드]

# 패턴
access-list 10 permit [예외 대상]
access-list 10 deny   [차단 대상]
access-list 10 permit any

# 핵심 규칙
위에서부터 순서대로 검사 먼저 매칭되면 끝 마지막에 implicit deny 존재 (자동 차단)
```
## 3. Extended ACL (100~199)
```
# 기본 구조
access-list [번호] permit|deny [프로토콜] [출발지] [목적지] [포트]

# 패턴
access-list 100 permit ip [출발지] [허용 목적지]
access-list 100 deny   ip [출발지] any
access-list 100 permit ip any any
```
## 4. 방향 선택 공식
✔ Standard ACL : 목적지 가까운 곳 (보통 out)
✔ Extended ACL : 출발지 가까운 곳 (보통 in)

## 5. in / out 개념
```
# in : 인터페이스로 들어오는 트래픽 검사
interface f0/0
 ip access-group 10 in

# out :  인터페이스에서 나가는 트래픽 검사
interface s1/0
 ip access-group 10 out
```


## 6.전체 실전 패턴

```
# ACL 생성
access-list 10 permit [예외]
access-list 10 deny   [차단 대상]
access-list 10 permit any

# 인터페이스 적용
interface s1/0
 ip access-group 10 out
```

## 7. 한 줄 공식
ACL = (허용 → 차단 → 전체허용) + (인터페이스 + 방향)

## 8. 실습 기준 핵심 정리
Standard ACL → out 방향 → 인터넷쪽 인터페이스    
Extended ACL → in 방향 → 사용자쪽 인터페이스    

## 9. 조심
+ deny 먼저 쓰면 예외 적용 안됨
+ permit any 안 쓰면 전체 차단됨
+방향(in/out) 반대로 하면 결과 틀림
+ Standard ACL은 목적지 조건 사용 불가

0. [프로토콜] / [출발지] / [목적지] / [포트] / [동작]

http      : 80           domain  : 53                   
https/ssl : 443          tftp    : 69
telnet    : 23           bootps  : 67
ssh       : 22           bootpc  : 68
ftp       : 21           ntp     : 123
ftp-data  : 20           snmp    : 161
smtp      : 25           syslog  : 514
pop3      : 110
mysql     : 3306

프로토콜 = 통신 방식 (큰 분류) => 건물 (TCP / UDP / ICMP)
포트 = 서비스 번호 (세부 식별) => 방 번호 (80번, 21번 등) : 서비스 이름 나오면 → 포트 사용

서비스 나오면 → TCP + 포트
Ping 나오면 → ICMP + 포트 없음
전체 허용 → IP

access-list 100 deny tcp ... eq 80
access-list 100 deny icmp ...
access-list 100 permit ip any any

| 문제 문장  | 프로토콜 | 포트 |
| ------ | ---- | -- |
| 웹 접속   | TCP  | 80 |
| FTP 접속 | TCP  | 21 |
| Telnet | TCP  | 23 |
| Ping   | ICMP | 없음 |


| 프로토콜 | 의미   | 포트 사용 여부 |
| ----- | ----------| -------- |
| TCP   | 연결형   | ⭕ 사용     |
| UDP  | 비연결형 | ⭕ 사용     |
| ICMP | ping      | ❌ 없음     |
| IP     | 전체       | ❌ 없음     |

정렬기준:

예외 먼저
작은 범위 먼저

1. 예외 (permit)
2. 특정 host 차단
3. subnet 차단
4. 다른 조건 차단
5. 전체 허용

1. ICMP 예외 (서버 → 외부)
2. ICMP 차단 (외부 → 서버)
3. host 단위 (13.13.20.2)
4. subnet 단위 (13.13.20.0/24)
5. 다른 subnet (13.13.10.0/24)
6. 전체 허용

방향:
in  = 인터페이스로 "들어올 때" 검사 -> 외부 → R3 들어올 때
out = 인터페이스에서 "나갈 때" 검사 -> R3 → 외부 나갈 때

[외부] ───▶ (R3 S1/1) ───▶ [R3 내부]
                   ↑ in 검사
                   ↓ out 검사

ip access-list = ACL "만드는 명령어"
ip access-group = ACL "붙이는 명령어" <- 반드시 인터페이스에 붙여야 동작함 (무조건 필요)

| 명령어               | 역할            | 어디서 씀    |
| ----------------- | ------------- | -------- |
| `ip access-list`  | ACL 정의(내용 작성) | 글로벌 모드   |
| `ip access-group` | 인터페이스에 적용     | 인터페이스 모드 |

 ① R3는 외부에서 오는 트래픽 중 출발지 네트워크가 ‘13.13.10.0/24’인 트래픽이 내부 로컬 네트워크 ‘13.13.30.0/24’로 Telnet 접근을 차단하여라. 
```
# [프로토콜] / [출발지] / [목적지] / [포트] / [동작] : Telnet 차단
# TCP / 13.13.10.0/24 / 13.13.30.0/24 / 23 / deny
```
 ② R3에 위치한 HTTP/FTP 서버는 외부로부터 ICMP 공격을 받고 있는 상태이다. 외부로 부터 수신하는 ICMP를 차단하여 공격을 방지하여라. 단, 서버는 외부로 Ping이 가능해야 한다.
```
# [프로토콜] / [출발지] / [목적지] / [포트] / [동작] : ICMP 차단 + 예외
# ICMP / any / 13.13.30.0/24 / - / deny
# ICMP / 13.13.10.0/24 / any / - / permit
```
 ③ R3는 외부에서 오는 트래픽 중 출발지 네트워크가 ‘13.13.20.0/24’인 트래픽이 내부 로컬 웹서버 ‘13.13.30.3’에 접근하는 것을 차단하여라.
```
# [프로토콜] / [출발지] / [목적지] / [포트] / [동작] : 웹차단
# TCP / 13.13.20.0/24 / 13.13.30.3/ 80 / deny

```
 ④ R3는 외부에서 오는 트래픽 중 출발지 네트워크가 ‘13.13.20.2’인 트래픽이 내부 로컬 FTP 서버 ‘13.13.30.3'에 접근하는 것을 차단하여라.
```
# [프로토콜] / [출발지] / [목적지] / [포트] / [동작] : FTP 차단 (특정 host)
# TCP / 13.13.20.2/ 13.13.30.3 / 21 / deny

```
 ⑤ R3는 위에 조건을 제외한 나머지 트래픽은 허용한다. 
```
# [프로토콜] / [출발지] / [목적지] / [포트] / [동작] : 나머지 허용
# IP / any / any / - / permit

```
 - ACL 항목을 R3 Serial 1/1 에 적용하여라.

# 풀이
<img width="964" height="365" alt="image" src="https://github.com/user-attachments/assets/314de71d-8f06-493a-92ac-0d1952f90c51" />

```
# extended ACL
# 서버 -> 외부 ping 허용 (예외먼저)
access-list 110 deny tcp 13.13.10.0 0.0.0.255 13.13.30.0 0.0.0.355 eq 23
access-list 110 deny icmp any host 13.13.30.3 echo
access-list 110 deny tcp 13.13.20.0 0.0.0.255 host 13.13.30.3 eq 80
access-list 110 
```
 @ 설정 완료 이후, 정보 확인

- PC0에서 CMD 이후, telnet 13.13.30.1로 텔넷 접속이 안되는지 확인하여라.
- PC0에서 CMD 이후, ping 13.13.30.3으로 Ping이 안되는지 확인하여라.
- PC2에서 Web Browser 실행 이후, www.soldesk.com 접속이 안되는지 확인하여라.
- PC2에서 CMD 이후, ftp 13.13.30.3 접속이 안되는지 확인하여라.
- 그외 나머지 트래픽들은 접근 되어야 한다.
